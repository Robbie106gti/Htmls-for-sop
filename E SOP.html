House keeping: CADCode File Maintenance SOP

  1. Maintaining the Panel Saw Programs. File names are created with the batch number which is the date of the month and the material name. The panel saw operator download one batch at the time. To avoid contamination with files from previous month delete the files that we don’t need any more. Twice a month (before you start working on batch that number is higher than 15 or is for the next month) delete the files that are for batches 1 to 15 (when you start a new month) or batches from 16 to 31 when you start the second half of the month. The files on the server which have to be deleted are on:
    Q:\Q5 CADCODE\Saw

    At the end of each month (before a batch for the next month has been started) the Panel Saw programs that are located on the CADCode computer have to be archived. They are located on:
    C:\CADCode\Saw

    In that folder delete all empty subfolders, select all files for the month with extension “NC” right mouse button click on a file from the selection, from the context menu select WinZip -> Add to WinZip file … , in the field “Add to archive” after the path “C:\CADCode\Saw\” write a name for the archive in format “YYYY.MM.zip” and hit the button “ Add”. Then delete select all files for the month with extension “NC”.

  2. Maintaining the program files for Rover B. File name for programs for Rover B are unique for each part. They are organized in folders with batch name. The CNC operator downloads one batch at the time. If programs for more than one month get together in folder with same name the excess of files’ quantity will slow down the computer on Rover B. Delete the folders with the programs for Rover B following the same rule as the programs for the panel saw. Twice a month (before you start working on batch that number is higher than 15 or is for the next month) delete the files that are for batches 1 to 15 (when you start a new month) or batches from 16 to 31 when you start the second half of the month. The files on the server which have to be deleted are on:
    Q:\Q5 CADCODE\Rover B


    At the end of each month (before a batch for the next month have been started) the Rover B programs that are located on the CADCode computer have to be archived. They are located on:
    C:\CADCode\Rover B

    In that folder select all folders for the month with extension right mouse button click on a file from the selection, from the context menu select WinZip -> Add to WinZip file … , in the field “Add to a rchive” after the path “C:\CADCode\Rover B\” write a name for the archive in format “YYYY.MM.zip” and hit the button “Add”. Then delete select all folders for the month. Then move the new created archive to:
    C:\CADCode\BackUps\Rover B

  3. Maintaining the program files for Rover A. File names for rover A are unique. They are organized in folders with batch name. The CNC operator downloads one batch at the time. If programs for more than one month get together in folder with same name the excess of files’ quantity will slow down the computer on Rover A. Delete the folders with the programs for Rover A following the same rule as the programs for Rover B. Twice a month (before you start working on batch that number is higher than 15 or is for the next month) delete the files that are for batches 1 to 15 (when you start a new month) or batches from 16 to 31 when you start the second half of the month. The files on the server which have to be deleted are on:
    Q:\Q5 CADCODE\Rover A

    C:\CADCode\Nest Code

    In that folder select all folders for the month with extension right mouse button click on a file from the selection, from the context menu select WinZip -> Add to WinZip file … , in the field “Add to a rchive” after the path “C:\CADCode\Nest Code\” write a name for the archive in format “YYYY.MM.zip” and hit the button “Add”. Then delete select all folders for the month. Then move the new created archive to:
    C:\CADCode\BackUps\Rover A

  4. Maintaining the label files for Rover A. The label files for rover A have extension “MDB”. Each file represents a batch. Each batch file contains database of label information organized in tables by material and date. It is possible the files for more than two months to merge and become too oversized and the list with materials might confuse the CNC operator. At the end of each month (before a batch for the next month have been started) the Rover A label files that are located on the CADCode computer have to be moved to a separate folder. They are located on:
    C:\CADCode\Label Data

  5. Maintaining the label files for Rover B. The label files for rover B have extension “MDB”. Each file represents a batch.
