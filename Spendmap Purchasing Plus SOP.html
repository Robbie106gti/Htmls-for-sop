Notes about Spendmap:
  • Items with an asterix are preferred to order items.
  • BMT = the slide for DPBM and DMBM drawer boxes. e.g. BMT550
  • BMTB = the slide for DPBM and DMBM drawer boxes. e.g. BMTB550
  • Pine = PI
  • Knotty material = K eg Knotty pine = KPI
  • Black = BL
  • When a note pops up saying 1 of 2, order the same code name followed with a number 2 to get all parts required.
  • When a note pops up saying 1 of 2? order the same code name followed with a number 2 if you need the related part parts.
  • When ordering items that are a left or right, add a L or R after the code.
  • Eg LS-MC-CHL for a left or LS-MC-CHR for a right.
  • With moldings, specific desired lengths must be noted in notes area in the line directly below original notes. e.g.’s
  • 1x8’ (which means 1 length at eight feet long)
  • 1x8’ + 1x4’ + 1x10’
  • INS (installed) and SO (Supply only) are not included in the codes on Spendmap. Eg. Code in Quoin MSC-GR-INS = code in Spendmap GRBL
  • SCM. Order FCSM. Note that 1 x FCSM = 2 SCM
Ordering codes in purchasing plus:
Sheet Materialsbeing ordered match our material names SOP

Note:

  • That with horizontal materials, there will be no space in purchasing plus. Eg BA H will be BAH, etc.
  • Pine = PI
  • WO is all quarter sawn.
  • If a material is required with a core other then our standards, add it with the thickness. Eg WO-19MDF-WO
Thickness are:

5/8” = 16
DA Baltic Birch Ply = 15
¾” = 19
½”= 13
3/8” = 9

Sheet size = 4’ x 8’ except the DA ply materials that are 5’ x 5’. If another size is wanted, it is ordered by adding the size following the std code

eg. PT-16-PT is automatically a 4x8

PT-16-PT510 is 5’wide x 10’ long. (Unfortunately we don’t have enough room too make it 5X10.)

How to create a requisition in Spendmap:
  • Open Spendmap “Requisition Version” using predetermined name and password.
  • Select "Item Req." at the top left of the screen.
  • A window titled "Item requisition batch" will appear. Click "Add".
  • Enter Dept (left click on? if needed) and enter or click "OK".
  • Add (left click for entire list, right click for search format) and OK.
  • Must fill in Job #, date required.
  • Add more items to this dept if needed, then click "back".
  • Select Add to order for a different dept. or Done if finished.
  • Save the requisition to Z:/Requisitions, saving by date. Add a 'b' or 'c' etc if this is used more than once in a day.
How to turn requisitions into PO’s in Spendmap:
  • Select "Requisition" on the "Purchasing/Main" window.
  • Select Approve "Items" from the top menu.
  • Select requisition you wish to approve.
  • Select all items you wish to approve, then click "Accept", then "OK".
  • Select "OK" on the popup menu.
  • Select another item if more are needed.
    Select "back" when done.
  • Select "Build items" from the top menu in "Requisitioning".
  • Select "OK", then "OK" again.
  • Select "Purchase" from the "Requisitioning" window.
  • On the Menu bar, Go to Purchase Order > Staging Batch > View/process staging batch.
  • Select any item you wish to put on hold (e.g. Blum items), or need to adjust by double clicking. If the status column has a checkmark, they are good to go.
  • Select Done.
  • Select OK until back on the purchasing/main page.
  • Select: "New PO’s" (top left of purchasing/main screen)
  • Select Supplier you wish to create PO for by double clicking on it.
  • Tab through all fields until you move to the next page.
  • Select and validate any item that needs it.
  • Select the last item on the list.
  • In the notes section, below any notes, add the appropriate boilerplate. This is usually boilerplate 1.
  • Select OK.
  • Click "Back"
  • Then "OK"
  • Select OK again
  • Select the next PO you want to process or select Done when completed.
  • Select OK.
  • Print.
  • Yes
  • Select OK, until complete
  • Look at all printouts:
  • Check for any notes and see if they effect anything.
  • Check for items we have in stock.
  • Fax PO’s to Suppliers.
How to Receive/close an item/PO in Spendmap:
  • You must be in Spendmap “Purchasing/Main”
  • Select either receive option.
  • Select "Receive" from the top menu.
  • Enter PO number into the "P.O. number" field, then select "OK".
  • Double click on the item you wish to receive.
  • Enter the amount received into the "Quantity received" field, and push enter or click "OK".
    (If the amount changed from the original number (e,g, DWD), enter the new number, check the override box. Then select the view PO option and print out a new copy.)
  • Select OK.
  • Select next item you wish to close or select back.
  • Enter the next PO number or select cancel.
  • Select purchase to return to the main screen.
How to create New PO’s in Spendmap:
  • You must have SpendMap-Purchasing/Main open.
  • Select "New PO" from the top menu.
  • Add.
  • Select Supplier and tab through all fields until you move to the next page.
  • Enter the item code or right click on the ? too search for the item.
  • Tab through the remaining fields, typically filling in the Job# and delivery date.
  • Select OK
  • For the last item for a particular supplier, add the boilerplate 1 note.
  • Select "Add" to add more if needed, or "Back" if completed.
  • Select "OK", then "Done", then "OK" again
  • Click "Print". Continue through the prompts, pressing "Yes" or "OK" until the document prints.
  • Look at all printouts:
  • Check for any notes and see if they effect anything.
  • Check for items we have in stock.
  • Fax PO’s to Suppliers.
How to modify a PO in Spendmap:
  • You must be in SpendMap-Purchasing/Main open.
  • From the menu bar, go to "Purchase Order > Purchase Orders > Modify a processed P.O. > Modify a processed P.O. > Simple (modify one P.O. line at a time on-line)".
  • Enter the PO number into the "P.O. number" field, then click "OK".
  • Double click on item you wish to modify.
  • Modify appropriate line(s) and select OK.
  • Repeat process for other items or choose Back if done.
  • Select OK because you usually want a new copy.
  • Select Print
  • OK
  • Enter new PO and repeat or select cancel if done.
